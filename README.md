# 家庭电子菜单微信小程序

一个原生微信小程序 MVP，包含顾客端点餐和服务侧管理。

## 已实现功能

- 顾客端
  - 菜品展示
  - 加减点餐数量
  - 填写备注
  - 提交订单
  - 重置当前点餐
- 服务侧
  - 查看点餐请求
  - 接单、完成订单
  - 清空订单
  - 新增和编辑菜品
  - 菜品上下架
  - 恢复默认菜品

## 运行方式

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择本目录：`/Users/zeru/Documents/codes/family_menu`。
4. AppID 可以先使用测试号或填你自己的正式小程序 AppID。
5. 导入后点击“编译”。

当前 `project.config.json` 里的 `appid` 是 `touristappid`，正式调试前建议替换成你的小程序 AppID。

## 当前数据方案

当前版本用 `wx.setStorageSync` 存储菜品和订单，适合本机演示和流程确认。

注意：本地存储只在当前设备和当前开发环境内可见。真正让“顾客手机下单，服务侧手机接收”需要后端同步能力。

## 你需要准备的基建

### 必需

- 微信公众平台账号：注册小程序并拿到 AppID。
- 微信开发者工具：用于导入、预览、真机调试和上传代码。
- 小程序成员权限：开发者、体验者、管理员按实际人员配置。

### 多设备接单必需

任选一种后端方案：

- 微信云开发 CloudBase：推荐给这个项目的第一版，配置简单，适合菜品、订单、状态流转。
- 自建 HTTPS API：需要域名、备案或合规域名配置、TLS 证书、服务端、数据库。

需要的数据表或集合：

- `dishes`：菜品名称、分类、价格、描述、是否上架。
- `orders`：订单菜品、数量、总价、备注、状态、创建时间。

### 上线前需要

- 合法服务类目和小程序基础信息。
- 小程序隐私协议配置。
- 服务器域名配置，如果使用自建后端。
- 体验版测试账号和测试流程。
- 提交审核前的页面内容、按钮、业务说明检查。
